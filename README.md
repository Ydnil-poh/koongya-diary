# 📄 [PRD v4.2] 쿵야 다이어리 (Koongya Diary)

## 1. 프로젝트 개요
- 사용자의 사유를 '쿵야' 캐릭터의 성장과 결합한 AI 마인드셋 다이어리.
- 3x3 아이소메트릭 그리드 기반의 픽셀 아트 UI.

## 2. 쿵야 라인업 (최종)
1. **양파**: 맑눈광 (말투: ~다쿵)
2. **주먹밥**: 안습/친절 (말투: ~용)
3. **무시**: 무심/락스타 (말투: 단답형)
4. **반계**: 열정/긍정 (말투: 가보자고!)
5. **피망**: 힙스터/예술가 (말투: 영단어 혼용)
6. **셀러리**: 허세 매니저 (말투: 우아한 척)
7. **마늘**: 독설/최종 교정자 (말투: ~마늘!) *파셀리 대용*

## 3. 성장 로직 (5-3-1 로직)
- **1단계**: 대화 5회 후 회고/진화 가능
- **2~4단계**: 대화 2~3회 후 회고/진화 가능
- **5단계**: 대화 1회 후 **최종 윤문 및 졸업 제의**

## 4. 핵심 장치
- **5단계 특수 로직**: AI가 지금까지의 대화를 요약하고, 사용자의 일기를 최종 윤문/교정 제안.
- **졸업 시스템**: 최종 일기 작성 후 폴라로이드 사진(핵심 질문 포함) 생성 및 아카이브 저장.

## 5. 협업 구조 (비개발자 + 웹디자이너)

### 파일 역할 분리
- `js/app.js`: 화면 이벤트 바인딩과 사용자 플로우 중심(진입점).
- `js/services/clients.js`: Supabase/Gemini 초기화 및 AI 폴백 처리.
- `js/data/koongyas.js`: 캐릭터 데이터(텍스트/성격).
- `js/ui/common.js`: 토스트, 로딩 오버레이, 정원 렌더링 등 공통 UI 함수.
- `css/style.css`, `assets/images/**`: 디자이너(배우자)가 주로 수정하는 영역.

### 추천 브랜치 전략
1. `main`: 배포 안정 버전.
2. `feat/ui-*`: 디자인/스타일/에셋 변경.
3. `feat/app-*`: 기능 변경.
4. PR 단위로 리뷰 후 `main` 병합.

### 충돌 줄이는 규칙
- 디자이너 작업은 우선 `css/`, `assets/` 안에서 진행.
- 기능 수정 시 `js/services`, `js/data`, `js/ui` 단위로 나눠서 커밋.
- 한 PR에서는 “기능”과 “디자인”을 가능하면 분리.
